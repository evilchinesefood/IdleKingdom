## Tasks — AUTO-IMPLEMENTED this session

### 1. [CRITICAL] Remap the Web Awesome brand ramp from blue to gold

**Location:** `Source/Styles/WaTheme.css`
**Problem:** Only `--wa-color-brand-fill-loud/-fill-quiet/-on-loud` were remapped. `variant="brand"` components derive border/outline/hover/on-quiet from the `--wa-color-brand-05…95` ramp, which WA chains to `--wa-color-blue-*`. Result: blue borders/outlines/hover on every primary button, the tooltip, and in-progress callouts.
**Fix:** Define the full `--wa-color-brand-05…95` ramp (+ `--wa-color-brand`, `-on`) as gold tints in `.kingdom-theme`; WA derives all semantic brand tokens from them.
**Verification:** `grep wa-color-brand Source/Styles/WaTheme.css` shows the 11-step ramp; no blue on any brand control in browser.

### 2. [CRITICAL] Brand button text contrast (parchment/white on gold = 1.99:1)

**Location:** `Source/Styles/WaTheme.css`
**Problem:** `--wa-color-brand-on-loud` was `parchment`; WA's default is `white`. Both fail WCAG on the gold accent fill.
**Fix:** `--wa-color-brand-on-loud: var(--ink)` (ink on gold = 6.5:1).
**Verification:** accent buttons show dark ink labels.

### 3. [CRITICAL] Link-delete "×" renders as a solid black blob

**Location:** `Source/Styles/Graph.css` (classes emitted in `GraphView.js:298-322`)
**Problem:** `.link-delete-hit` circle and `.link-delete` text had no CSS → circle defaults to black fill painting over the link midpoint.
**Fix:** add `.link-delete-hit{fill:var(--panel);stroke:var(--line);cursor:pointer}` and `.link-delete{fill:var(--bad);font-size:14px;font-weight:700;pointer-events:none}`; bump hit `r` 10→13 in GraphView.
**Verification:** tap a link → a small parchment circle with a red × at the midpoint.

### 4. [CRITICAL] Service worker breaks the PWA offline + corrupts sub-resource MIME

**Location:** `ServiceWorker.js`
**Problem:** SHELL caches none of the WA component/chunk files; cache-first + `.catch(()=>Index.html)` returns HTML for failed `.js`/`.css`/`.woff2` → "expected JavaScript but got text/html" and dead dialogs/tabs offline.
**Fix:** runtime-cache successful same-origin GETs (so WA components populate on first online use); gate the `Index.html` fallback to `request.mode === "navigate"` only; bump cache `v11→v12`.
**Verification:** offline reload still renders WA components; non-navigation failures don't return HTML.

### 5. [CRITICAL] No FOUC cloak for undefined WA elements

**Location:** `Source/Styles/Reset.css`
**Problem:** before `customElements.define`, every `wa-*` is unstyled inline text → flash/CLS on cold load.
**Fix:** `wa-*:not(:defined){visibility:hidden}` (and `display:none` for `wa-dialog`).
**Verification:** no raw-text flash before WA upgrades.

### 6. [CRITICAL] Stylesheet load order (WaTheme before Theme)

**Location:** `Index.html`
**Problem:** `WaTheme.css` (consumes `--gold` etc.) loaded before `Theme.css` (defines them). Survives via lazy `:root` resolution but is brittle.
**Fix:** load `Theme.css` before `WaTheme.css`.
**Verification:** view-source order: Fonts → Theme → Reset → WaTheme → Layout → Graph.

### 7. [HIGH] SVG nodes don't scale with zoom (text/icon/offsets fixed px)

**Location:** `Source/UI/GraphView.js:346-463` (`_drawNode`)
**Problem:** box dims used `*v.scale` but interior offsets/foreignObject/fonts were constant → text spills, icon detaches at any non-100% zoom.
**Fix:** render each node as one `<g transform="translate(p) scale(v.scale)">` with children in local unscaled coords; pixel-identical at scale 1, uniform at any zoom. Also makes port draw match the graph-space hit-test.
**Verification:** zoom in/out — node contents scale uniformly; 281 tests still green.

### 8. [HIGH] Link hit-test desyncs from drawn curve when zoomed

**Location:** `Source/UI/GraphView.js:93-119` (`hitLink`)
**Problem:** hit Bézier built in graph space; draw builds it in screen space; `linkBezier`'s `max(40,…)` clamp isn't scale-invariant → clicks miss at scale≠1.
**Fix:** compute `hitLink` in screen space (mirror `_draw`: `graphToScreen` endpoints, pointer→screen, `tol` in screen px).
**Verification:** zoom, then click a link curve — reveal/delete fire on the visible line.

### 9. [HIGH] 44px tap-target regression after wa-button migration

**Location:** `Source/Styles/Layout.css`
**Problem:** `min-height:var(--tap)` only existed inside `@media(max-width:720px)` and was set on the host, not `::part(base)`. Desktop wa-buttons were 43px (default) / 38px (`size=s`).
**Fix:** global `wa-button::part(base){min-height:var(--tap)}`.
**Verification:** all buttons ≥44px tall on desktop and mobile.

### 10. [HIGH] `.factory-legend` overlaps the bottom-sheet on mobile

**Location:** `Source/Styles/Layout.css` (`@media max-width:720px`)
**Problem:** legend pinned bottom-left (`z-index:30`) sits under the inspector sheet (`bottom:0`, `z-index:40`).
**Fix:** `.factory-legend{display:none}` in the ≤720px block.
**Verification:** no legend/sheet overlap on a narrow viewport.

### 11. [HIGH] "Always show rates" toggle is dead

**Location:** `Source/Styles/Layout.css` + `Source/UI/Prefs.js`
**Problem:** `.show-rates` class is toggled but has no CSS; `formatRate` always returns a string so rates always render → the pref no-ops.
**Fix:** add `.hud:not(.show-rates) .hud-cur .rate{display:none}` + `@media(hover:hover){.hud:not(.show-rates) .hud-cur:hover .rate{display:inline}}`; default `alwaysShowRates:true` (preserves current always-on default & is mobile-safe; toggling off declutters to hover-reveal).
**Verification:** toggle in Settings hides/shows the per-currency rate.

### 12. [HIGH] Victory dialog is Escape-dismissible (epilogue lost forever)

**Location:** `Source/UI/Victory.js`
**Problem:** `onWaHide: onClose` → Escape acks victory; it never reappears (overlay light-dismiss is off, so Escape/× are the leak).
**Fix:** make Victory acknowledged-only — `onWaHide` calls `preventDefault()` (WA reopens), acknowledge only via the "Continue the Reign" button.
**Verification:** Escape on the victory screen keeps it open; only Continue closes+acks.

### 13. [HIGH/MED] Additive a11y: labels, live regions, safe-area

**Location:** `Source/UI/Hud.js`, `Source/UI/App.js`, `Source/Styles/Layout.css`
**Problem:** currency tags differ only by a small icon (no accessible name); errors aren't announced; iOS bottom sheets ignore safe-area + use `vh`.
**Fix:** `aria-label="<Currency>: <value>"` on each currency tag; `role="alert"` on the error host; bottom sheets `45vh→45dvh` + `padding-bottom: env(safe-area-inset-bottom)`. (Deliberately NOT adding `aria-live` to the currency row — per-second idle updates would spam screen readers; the `aria-label` lets users query the value on demand instead.)
**Verification:** screen-reader announces a named currency when focused; the error flash is announced; sheets clear the home indicator.

### 14. [LOW] Consistency + cleanup

**Location:** `Source/UI/App.js`, `Source/UI/NodeInspector.js`, `Source/Styles/Reset.css`, `Index.html`, `Source/Styles/Layout.css`
**Fix:** unify starved wording to "LOW" (legend + inspector match the node badge); remove dead `.modal` from the Reset user-select list; drop `maximum-scale=1` (canvas already owns gestures via `touch-action:none`); set `.exp-progress{--track-height:6px}` to match `.ni-cap`.

## Tasks — original deferred notes (for reference)

### 15. [HIGH] Dialog close lifecycle: drive `prop:open=false`, remove on `wa-after-hide`

Settings/OfflineSummary/Victory are torn from the DOM on close (hardcoded `prop:open:true` + add/remove) instead of animating out via `prop:open=false`; also a pre-define `prop:open` property-set race may stop OfflineSummary opening on a cold load. Needs a small overlay-render refactor + real-browser verification of WA open/close timing.

### 16. [HIGH] Anchored onboarding tooltips

`Tooltip.js` sets `data-anchor` but `App.js` never positions by it — every tip floats bottom-center and overlaps mobile sheets. Either implement `getBoundingClientRect`-based anchoring (with a connector) or drop the dead `anchor` field. Needs browser to place/verify.

### 17. [HIGH] Graph canvas keyboard accessibility (WCAG 2.1.1, Level A)

Node select/drag/connect, pan/zoom, link reveal/delete are pointer-only; no SVG element is focusable. Make nodes `tabindex`/`role=button`/`aria-label`; Enter to select, arrows to nudge, keyboard connect path. Large effort.

### 18. [MED] First-run guidance (blank canvas)

Empty start is by design, but add an on-canvas empty-state or a build-menu highlight while `seenGoldTip` is unset so new players know where to begin. Product/design call.

### 19. [MED] Link & node interaction discoverability

`.node-card` always shows `cursor:move` but first press only selects; links have no hover affordance and are the only place to delete a connection. Show `cursor:move` only on `.selected`; add hover thicken + a persistent mid-link affordance. Browser to tune.

### 20. [MED] `wa-select` controlled-value under live snapshots

Per-tick re-render re-asserts `prop:value`, which can reset an open dropdown; pre-define value race may blank the initial selection. Verify in browser; guard re-assert when focused/open, or set value as an attribute / after `whenDefined`.

### 21. [MED] Save badge

Always-on "saved" tag is clutter; failure tag (outlined danger) is low-emphasis. Verify `meta._saveStatus → snap.save.status` actually surfaces failure, then show transiently / emphasize failure. Product + verification.
